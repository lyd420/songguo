# 网文写作助手 - P0 用户故事

> 基于 Mike Cohn + Gherkin 格式编写，覆盖 PRD 中所有 P0 功能

---

## 用户画像定义

| 角色 | 描述 |
|-----|------|
| 新手作者 | 0-1本书，未签约或刚签约，需要模板和引导 |
| 成熟作者 | 3本以上，稳定更新，关注效率和质量诊断 |
| 平台作者 | 在点众平台投稿的所有作者 |

## User Story 001：新手使用结构模板开启第一章

- **Summary：** 让新手作者通过模板快速写出符合平台风格的开篇

### Use Case：
- **As a** 0-1本书的新手作者（未签约）
- **I want to** 根据热门题材选择结构模板
- **so that** 快速构建第一章的剧情框架并通过编辑审核

### Acceptance Criteria：
- **Scenario：** 首次使用系统的新手作者选择题材模板
- **Given：** 我是首次访问系统的新手作者
- **and Given：** 系统显示题材选择界面（玄幻/都市/言情/悬疑）
- **and Given：** 我选择"都市"题材
- **When：** 点击"开始创作"按钮
- **Then：** 系统推荐 3 个热门开头模板（经典打脸/悬念揭晓/情感爆发）
- **and Then：** 点击模板后，编辑器预填充该结构的示例内容
- **and Then：** 示例内容可被一键清空，让我输入自己的内容

## User Story 002：作者管理章节层级结构

- **Summary：** 让作者通过卷/章结构管理长篇小说

### Use Case：
- **As a** 连载长篇小说的平台作者
- **I want to** 创建和管理卷/章层级结构
- **so that** 清晰组织超过 100 章的长篇小说

### Acceptance Criteria：
- **Scenario：** 作者创建第一卷并添加章节
- **Given：** 我已登录系统并进入作品管理页面
- **and Given：** 当前作品没有任何卷章
- **When：** 点击"创建第一卷"并输入卷名（如"第一卷：潜龙在渊"）
- **Then：** 系统创建第一卷并在左侧导航显示
- **and Then：** 点击"新建章节"输入章节名后，章节显示在该卷下
- **and Then：** 可通过拖拽调整章节顺序

## User Story 003：实时字数统计与目标提醒

- **Summary：** 让作者实时了解写作进度，达成日更目标

### Use Case：
- **As a** 需要在点众平台日更的平台作者
- **I want to** 实时查看当前章节字数和日更目标完成度
- **so that** 确保达到 4000-6000 字的日更要求

### Acceptance Criteria：
- **Scenario：** 作者写作时查看字数统计
- **Given：** 我正在编辑器中写作
- **and Given：** 我的日更目标设置为 4000 字
- **When：** 输入文字时
- **Then：** 编辑器底部实时显示当前章节字数（如"本章：1,234 字"）
- **and Then：** 显示今日累计字数（如"今日：2,567 / 4000 字"）
- **and Then：** 当日更目标达成时，显示视觉庆祝提示（如进度条变绿）

## User Story 004：自动保存防丢稿

- **Summary：** 保护作者辛苦创作的稿件不丢失

### Use Case：
- **As a** 长时间写作的平台作者
- **I want to** 写作内容自动保存
- **so that** 即使意外关闭浏览器也不会丢失稿件

### Acceptance Criteria：
- **Scenario：** 作者在写作过程中系统自动保存
- **Given：** 我正在编辑器中写作且内容已变更
- **and Given：** 自上次保存已超过 30 秒
- **When：** 系统触发自动保存
- **Then：** 编辑器角落显示"已保存"状态提示
- **and Then：** 若我意外刷新页面，重新打开后内容自动恢复
- **and Then：** 切换章节时如有未保存内容，提示确认保存

## User Story 005：本章 AI 结构诊断

- **Summary：** 让作者了解当前章节的结构完整性

### Use Case：
- **As a** 想提升写作质量的成熟作者
- **I want to** 对本章进行 AI 结构诊断
- **so that** 了解起承转合的完整度和改进建议

### Acceptance Criteria：
- **Scenario：** 作者完成一章后请求结构诊断
- **Given：** 我已完成一章的写作（字数 > 500 字）
- **and Given：** 当前章在编辑器中打开
- **When：** 点击"本章诊断"按钮
- **Then：** AI 面板显示结构诊断结果（起/承/转/合 进度条）
- **and Then：** 显示字数分布分析（开头/中间/结尾比例）
- **and Then：** 提供 2-3 条具体的结构优化建议

## User Story 006：情绪曲线可视化

- **Summary：** 让作者直观了解章节的情绪起伏

### Use Case：
- **As a** 注重读者阅读体验的成熟作者
- **I want to** 查看本章的情绪曲线分析
- **so that** 确保情绪有起伏，避免平铺直叙

### Acceptance Criteria：
- **Scenario：** 作者查看章节情绪曲线
- **Given：** 我已完成一章的写作
- **and Given：** 当前章在编辑器中打开
- **When：** 点击"情绪分析"标签
- **Then：** 显示 SVG 情绪曲线图（X轴：章节进度，Y轴：情绪强度）
- **and Then：** 标注情绪高点（爽点）和低点（虐点）位置
- **and Then：** 显示"节奏过快/过慢"的警示区域

## User Story 007：卡文检测与急救

- **Summary：** 在作者卡文时温和介入，提供续写建议

### Use Case：
- **As a** 正在写作但突然卡文的平台作者
- **I want to** 在卡文时获得温和的提醒和建议
- **so that** 突破写作瓶颈继续日更

### Acceptance Criteria：
- **Scenario：** 作者卡文 5 分钟后系统介入
- **Given：** 我正在编辑器中写作
- **and Given：** 已有内容字数 > 200 字
- **and Given：** 我已连续 5 分钟无键盘输入
- **When：** 系统检测到卡文状态
- **Then：** 右下角显示温和提示（"需要一点灵感吗？"）
- **and Then：** 点击后提供选项：1)继续思考 2)查看模板建议 3)切换章节
- **and Then：** 提示 10 秒后自动消失，不打断写作心流

## User Story 008：快捷输入标点符号

- **Summary：** 让作者快速插入网文常用标点

### Use Case：
- **As a** 追求写作效率的平台作者
- **I want to** 一键插入网文常用标点（【】（）……）
- **so that** 减少标点输入时间，专注创作

### Acceptance Criteria：
- **Scenario：** 作者使用快捷工具栏插入标点
- **Given：** 我正在编辑器中写作
- **and Given：** 快捷工具栏显示在编辑器上方
- **When：** 点击工具栏的"【】"按钮
- **Then：** 在光标位置插入"【】"且光标定位在两符号之间
- **and Then：** 支持快捷键（如 Ctrl+1）快速插入【】
- **and Then：** 同样支持（）、……、——等常用符号

## User Story 009：爽点/虐点标记

- **Summary：** 让作者标记章节中的关键情节点

### Use Case：
- **As a** 重视卡点设计的成熟作者
- **I want to** 在文中标记爽点或虐点位置
- **so that** 分析卡点分布，优化付费转化率

### Acceptance Criteria：
- **Scenario：** 作者标记文中爽点位置
- **Given：** 我正在编辑器中写作
- **and Given：** 已选中一段文字（20-200 字）
- **When：** 点击工具栏的"🔥爽点"按钮
- **Then：** 选中文段旁显示 🔥 标记
- **and Then：** 在章节概览中显示爽点位置（如第 3 段、第 7 段）
- **and Then：** 支持 💔虐点 同样标记方式
- **and Then：** 鼠标悬停标记显示标记的具体内容摘要

## User Story 010：主题切换护眼

- **Summary：** 让作者在不同光线环境下舒适写作

### Use Case：
- **As a** 长时间写作的平台作者
- **I want to** 切换浅色/深色主题
- **so that** 在不同光线环境下保护视力

### Acceptance Criteria：
- **Scenario：** 作者切换编辑器主题
- **Given：** 我正在编辑器中写作
- **and Given：** 当前为浅色主题（莫兰迪米白 #F7F5F3）
- **When：** 点击右上角主题切换按钮
- **Then：** 编辑器背景变为深色主题（莫兰迪深灰 #2D2A28）
- **and Then：** 文字颜色自动适配（深色主题下变为浅灰）
- **and Then：** 主题偏好保存到本地，下次打开自动应用

|---|------|---------|---------|-----------|--------|
| 002 | 章节管理 | 所有作者 | 核心功能 | 中 | P0-1 |
| 003 | 字数统计 | 所有作者 | 核心功能 | 低 | P0-1 |
| 004 | 自动保存 | 所有作者 | 风险规避 | 中 | P0-1 |
| 001 | 结构模板 | 新手作者 | 差异化价值 | 中 | P0-2 |
| 010 | 主题切换 | 所有作者 | 体验优化 | 低 | P0-2 |
| 008 | 快捷输入 | 所有作者 | 效率提升 | 低 | P0-2 |
| 005 | AI结构诊断 | 成熟作者 | 差异化价值 | 高 | P0-3 |
| 006 | 情绪曲线 | 成熟作者 | 差异化价值 | 高 | P0-3 |
| 007 | 卡文检测 | 所有作者 | 差异化价值 | 中 | P0-3 |
| 009 | 爽点标记 | 成熟作者 | 进阶功能 | 低 | P0-3 |

## 注意事项

1. **Story 005、006 涉及 AI 功能**，建议 V1 使用规则匹配实现，V2 再接入 LLM
2. **Story 007 卡文检测**，需设置"免打扰"选项，避免频繁打扰
3. **Story 009 爽点标记**，数据可用于后续"卡点分析"功能（P1）
